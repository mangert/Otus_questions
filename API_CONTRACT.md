# Контракт API

## `GET /questions`

Возвращает вопросы мини-анкеты.

- Успешный статус: `200 OK`.
- Формат ответа: `application/json`.
- Тело ответа соответствует типу `GetQuestionsResponse` из пакета
  `@mini-survey/shared`.
- Массив содержит от трёх до пяти объектов.
- Каждый объект содержит строковые поля `id` и `text`.

Пример успешного ответа:

```json
[
  { "id": "example-1", "text": "Первый вопрос" },
  { "id": "example-2", "text": "Второй вопрос" },
  { "id": "example-3", "text": "Третий вопрос" }
]
```

## `POST /answers`

Принимает и сохраняет одну отправку с ответами пользователя.

- Формат запроса: `application/json`.
- Тело запроса соответствует типу `SubmitAnswersRequest` из пакета
  `@mini-survey/shared`.
- Поле `answers` содержит массив объектов со строковыми полями `questionId` и
  `value`.
- После успешной проверки вся отправка добавляется в хранилище в памяти.
- Данные не сохраняются между перезапусками backend.
- Успешный статус: `201 Created`.
- Успешный ответ не содержит тела и соответствует типу
  `SubmitAnswersResponse`.

Пример запроса:

```json
{
  "answers": [
    { "questionId": "example-1", "value": "Первый ответ" },
    { "questionId": "example-2", "value": "Второй ответ" },
    { "questionId": "example-3", "value": "Третий ответ" }
  ]
}
```

### Ошибки запроса

`POST /answers` возвращает `400 Bad Request`, если JSON некорректен или тело
запроса не соответствует контракту. В частности, отклоняются:

- отсутствующее или пустое тело запроса;
- тело запроса, которое не является JSON-объектом;
- отсутствующее поле `answers`, значение не в виде массива или пустой массив;
- неполный набор: нет ответа хотя бы на один существующий вопрос;
- элемент `answers`, который не является объектом;
- отсутствующие поля `questionId` или `value` и их нестроковые значения;
- пустые или состоящие только из пробелов `questionId` и `value`;
- неизвестный или повторяющийся `questionId`.

Отклонённая отправка не добавляется в хранилище. Ответ имеет формат
`application/json` и соответствует типу `ApiError` из пакета
`@mini-survey/shared`:

```json
{ "error": "Описание ошибки" }
```

Точный текст `error` не является частью контракта, но всегда представляет
собой непустую строку.
